# VPet-Gaze 视线模块：相对最初版本的增量说明

本文档面向项目报告，说明相对**一开始几乎未做增强的初版**，本模块后来增加了什么、各自解决什么问题。  
路径：`VPet/VPet-Gaze/python/`。

---

## 1. 最初版本（基线）长什么样

初版思路很简单：

| 环节 | 初版做法 |
|------|----------|
| 检测 | MediaPipe Face Mesh 取虹膜，算左右眼平均的水平/纵向比例 `(raw_x, raw_y)` |
| 校准 | 启动后看屏幕中心约 2 秒，只估一个纵向 `baseline` |
| 映射 | 水平用固定经验区间线性拉伸；纵向用 `tanh(高增益 × delta)` |
| 滤波 | EMA（纵向跟手系数还更大） |
| 预览 | 小窗摄像头画面上画点（不是真实屏幕坐标） |
| 输出 | FastAPI `GET /gaze` → VPet C# 跟随 |

**典型问题**：上下乱跳或方向反、看屏幕边缘到不了、眨眼/噪声导致断流跳变、预览难以判断「准不准」。

## 2. 相对基线：增加了什么（总表）

| # | 增加内容 | 作用（解决什么） |
|---|----------|------------------|
| 1 | **全屏屏幕坐标预览** | 绿点画在真实屏幕 `(screen_x, screen_y)` 上，方便验证注视位置 |
| 2 | **9 点校准（3×3 网格）** | 覆盖中心、四边中点、四角，边缘也能学到映射 |
| 3 | **特征加厚（双眼/开合/脸偏移等）** | 比单靠平均比例更稳，减轻单眼噪声 |
| 4 | **3D 头姿（solvePnP）** | 区分「转头」与「转眼」，减少抬头低头当眼球动 |
| 5 | **头 + 眼加性融合** | 显式组合 `gaze ≈ head + eye`，几何上更合理 |
| 6 | **二次多项式 + Ridge 映射** | 用户自适应：特征 → 屏幕坐标，适应个体与摄像头差异 |
| 7 | **Kalman 滤波（常速度模型）** | 比 EMA 更兼顾跟手与抗抖 |
| 8 | **眨眼保持 / 噪声帧处理** | 闭眼时不把注视点打飞，保持上一帧有效输出 |
| 9 | **镜像水平符号修正** | 摄像头镜像后左右与屏幕一致（避免盯右飞左） |
| 10 | **独立模块 `gaze_model.py`** | 特征/融合/Ridge/Kalman 与服务入口解耦，便于维护与写报告 |

**刻意未接入的重方案**：GazeFollower、L2CS-Net 等大模型（可作对比实验，非当前运行路径）。

## 3. 各项增量说明（报告可展开）

### 3.1 全屏预览（交互与验证）

- **增加**：按主屏分辨率建画布；注视点按归一化屏幕坐标绘制；`F` 全屏 / `Esc` 窗口 / `C` 重校准 / `Q` 退出。
- **作用**：初版点画在摄像头小图上，无法直观对应「屏幕哪里」；全屏后可直接盯四角验收。

### 3.2 九点校准（替代「只校中心」）

- **增加**：中心 → 上排三点 → 左右 → 下排三点；每点 settle + 采集多帧特征。
- **作用**：初版只有纵向中性点，水平靠死经验范围、边缘无监督；九点后整块屏幕都有标定样本。

### 3.3 特征加厚

相对初版仅 `(mean_h, mean_v)`，现特征包括（见 `gaze_model.FEATURE_NAMES`）：

- 左右眼水平/纵向虹膜比、开合度  
- 虹膜相对脸中心偏移  
- 3D 头姿 `head_yaw/pitch/roll`  
- 眼球角 `eye_yaw/pitch`  
- 融合角与几何屏幕点 `fused_*` / `geo_*`  

- **作用**：给回归更多可用信号；睁眼程度还可过滤眨眼脏数据。

### 3.4 3D 头姿（solvePnP）

- **增加**：鼻尖/下巴/眼角/嘴角 6 点 + 标准 3D 脸模型 → `cv2.solvePnP`（可 Refine + 上一帧初值）；头姿 EMA 抑制帧间抖。
- **作用**：初版完全不建模头部；微抬头会被当成「上下看」，纵向容易崩。

### 3.5 头眼加性融合

- **增加**：`fused = head + eye`，再映射为几何点 `geo_sx/sy`（作为特征输入 Ridge）。  
  当前输出以 **Ridge 为主**（`GEO_BLEND = 0`），避免几何与回归符号打架导致来回晃。
- **作用**：把「头怎么转、眼怎么转」拆开再合成；比单纯把比例硬塞进 tanh 更可解释。

### 3.6 Ridge + 二次多项式映射

- **增加**：特征标准化 → 二次展开 → Ridge 最小二乘拟合屏幕坐标；校准阶段保存每帧样本（上百条级）。
- **作用**：替代初版固定区间 + 高增益 tanh；同一套模型适配不同人、不同坐姿与摄像头。

### 3.7 Kalman 滤波

- **增加**：状态 `[x, y, vx, vy]` 的常速度 Kalman；限速与温和 jump，平衡跟手与稳定。
- **作用**：替代初版「纵向高 EMA」——要么抖、要么拖、扫视跟不上。

### 3.8 稳定性工程细节

| 细节 | 作用 |
|------|------|
| 眨眼 hold 上一帧 | 避免 valid 断流后猛跳 |
| 左右眼纵向冲突时降权/对齐 | 减少半闭眼噪声 |
| `INVERT_HEAD_YAW` | 镜像画面下左右与屏幕一致 |
| 死区保持较小（约 0.02） | 过大死区会把点吸回中心再弹开（来回晃） |

## 4. 当前数据流（定稿）

```
摄像头（水平镜像）
  → MediaPipe Face Mesh（含虹膜）
  → 虹膜比例 / 开合 / 脸偏移
  → solvePnP → 3D 头姿（平滑）
  → eye 角 + 头姿 → 加性融合 → geo 特征
  → 拼成约 19 维特征
  → 九点校准采集 → Ridge(poly2) 拟合
  → Kalman 平滑
  → 全屏绿点预览 + GET /gaze → VPet 桌宠
```

运行：

```bash
cd VPet/VPet-Gaze/python
python gaze_server.py
```

快捷键：`F` 全屏 · `Esc` 窗口 · `C` 重新九点校准 · `Q` 退出。

API 仍兼容：`valid / calibrating / screen_x / screen_y / gaze_x / gaze_y / raw_x / raw_y / timestamp`。

## 5. 基线 vs 当前（对比小结）

| 维度 | 最初（未增强） | 当前 |
|------|----------------|------|
| 校准 | 仅中心纵向 | 9 点全屏网格 + 多帧采样 |
| 特征 | 2 维眼比例 | 眼 + 3D 头 + 融合几何等 |
| 头姿 | 无 | solvePnP + 平滑 |
| 映射 | 经验线性 + tanh | Ridge + 二次项（用户自适应） |
| 滤波 | EMA | Kalman |
| 预览 | 摄像头小窗 | 全屏真实屏幕坐标 |
| 主要体验问题 | 上下跳、到不了边、难验收 | 需九点校准；消费级摄像头精度仍有上限 |

## 7. 报告可用摘要（一段话）

> 相对最初仅用 MediaPipe 虹膜二维比例、中心纵向校准与经验 tanh/EMA 的方案，本模块增加了全屏屏幕坐标预览、九点用户校准、多维眼部特征、基于 solvePnP 的三维头姿估计、头眼加性融合、二次 Ridge 用户自适应映射以及 Kalman 平滑，在不引入额外深度视线大模型的前提下，提升了注视方向一致性、边缘可达性与预览可验证性，并继续通过原有 `/gaze` 接口驱动 VPet 桌宠跟随。
